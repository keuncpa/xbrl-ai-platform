# 삼일PwC XBRL 내용 정리 및 AI 자동화 프로젝트 추천

---

## 1. 삼일PwC XBRL 페이지 핵심 내용 정리

금융감독원의 재무공시 선진화 로드맵에 따라 상장사 재무제표의 XBRL 공시가 단계적으로 의무화되고 있다. 삼일PwC는 국내 회계법인 중 유일하게 XBRL 전문조직(XBRL Center)을 별도로 구성하여 모든 XBRL 프로젝트를 전담 수행하고 있으며, 자체 개발한 Robotic XBRL Tool을 통해 업무 효율화를 추진 중이다.

### 주요 서비스 흐름 (4단계)

| 단계 | 내용 |
|------|------|
| **1. 사전 설계** | 재무제표 분석, XBRL 표준 검토, 회사 고유 Taxonomy 설계 |
| **2. 공시 자문** | 재무제표 변화사항 Taxonomy 업데이트, 분기별 XBRL 파일 작성 |
| **3. 디지털 도구** | Robotic XBRL (DSD/ixd → Excel 변환, 시뮬레이션, 숫자 검증, 버전관리) |
| **4. 품질 검증** | Excel 버전과 XBRL 파일 간 대사를 통한 정확성 검증 |

### 삼일PwC XBRL 조직 구성

| 조직 | 역할 |
|------|------|
| **XBRL Center** | XBRL 자문 전담 부서. 표준 Taxonomy 기반 품질 관리 |
| **AC (Acceleration Center)** | XBRL 작성기 프로그램 작업 전담 |
| **AX Node** | XBRL 업무의 디지털화/자동화 (Robotic XBRL 고도화) |
| **CRS (Corporate Reporting Service)** | 회계기준 전문가 조직. Taxonomy 적용 consultation |

### Robotic XBRL Tool 주요 기능

- DSD 파일 및 XBRL 파일(ixd)을 Excel로 자동 변환
- Excel 기반 XBRL 표 설계 및 다양한 시뮬레이션
- 직관적 숫자 입력 및 검증 기능
- Excel 버전과 XBRL 파일(ixd) 간 대사(교차 검증)
- XBRL 파일 간 변화사항 자동 추출 (버전관리)

## 2. AI 자동화 프로젝트 추천 (자소서 어필 순서)

아래 프로젝트들은 **삼일회계법인 신입 회계사 자소서에 어필하기 좋은 순서**로 나열했다. 순위 기준은 삼일PwC가 실제 운영 중인 XBRL 업무와의 직접적 연관성, 실현 가능성, 그리고 신입이 독자적으로 시도했을 때의 차별화 효과를 종합 고려했다.

---

### 프로젝트 1. XBRL Taxonomy 자동 매핑 엔진

**방향성:** 재무제표 계정과목을 XBRL 표준 Taxonomy 요소에 자동 매핑하는 AI 모델 개발. 삼일PwC가 사전 설계 단계에서 수행하는 Taxonomy 설계 업무를 직접 자동화하는 프로젝트로, 해당 법인의 핵심 업무와 정확히 맞닿아 있다.

**구체적 구현:**
- 금감원 공시 표준 Taxonomy(KOR-IFRS Taxonomy) 데이터셋 구축
- NLP 기반 계정과목 텍스트 → Taxonomy element 유사도 매칭
- 과거 매핑 이력 학습을 통한 정확도 향상

**완성도 기준:**
| 수준 | 설명 |
|------|------|
| MVP (2~3주) | 표준 계정과목 100개 대상 규칙 기반 매칭 + 정확도 측정 |
| 중급 (1~2개월) | 임베딩 기반 유사도 검색 + 산업별 분류 모델 |
| 고급 (3개월+) | LLM 기반 맥락 이해 매핑 + 확장항목 자동 생성 제안 |

**자소서 어필 포인트:** 삼일PwC 페이지에서 "회사 고유의 Taxonomy 설계"를 핵심 서비스로 소개하고 있다. 이 업무를 AI로 자동화하려 시도했다는 것 자체가 해당 법인의 업무를 깊이 이해하고 있다는 증거가 된다.

---

### 프로젝트 2. XBRL 파일 자동 검증 및 오류 탐지 시스템

**방향성:** XBRL 공시 파일의 오류를 자동으로 탐지하고 수정안을 제시하는 검증 시스템. 삼일PwC가 "업계 최고의 Quality"를 강조하고 있으므로, 품질 검증 자동화는 해당 법인의 핵심 경쟁력 강화에 직접 기여한다.

**구체적 구현:**
- XBRL Validation Rule 기반 문법/구조 오류 자동 탐지
- Calculation Linkbase 기반 합산 불일치 검출
- 전기 대비 이상치 탐지 (비율 변동, 금액 급변 등)
- 동일 산업 내 타사 공시와 비교 분석

**완성도 기준:**
| 수준 | 설명 |
|------|------|
| MVP (2~3주) | XBRL 파일 파싱 + 기본 Validation Rule 체크 (합산 검증) |
| 중급 (1~2개월) | 전기 비교 이상치 탐지 + 오류 유형별 분류 리포트 |
| 고급 (3개월+) | ML 기반 이상 패턴 학습 + 수정 제안 자동 생성 |

**자소서 어필 포인트:** 삼일PwC가 "Excel 버전과 XBRL 파일 간 대사를 통한 정확성 검증"을 Robotic XBRL의 핵심 기능으로 소개하고 있다. 이 검증 프로세스를 AI로 고도화하는 프로젝트는 기존 도구의 확장 방향을 정확히 짚는 것이다.

---

### 프로젝트 3. 재무제표 → iXBRL 자동 태깅 도구

**방향성:** PDF 또는 Excel 형태의 재무제표를 읽어 iXBRL(Inline XBRL) 태그를 자동으로 부착하는 도구. 현재 수작업 비중이 매우 높은 XBRL 태깅 업무를 직접 자동화한다.

**구체적 구현:**
- PDF/Excel 재무제표에서 숫자·텍스트 항목 추출 (OCR + 테이블 파싱)
- 추출된 항목과 Taxonomy 요소 자동 매핑
- iXBRL 태그 자동 생성 및 HTML 파일 출력

**완성도 기준:**
| 수준 | 설명 |
|------|------|
| MVP (2~3주) | Excel 재무상태표 1개 양식 대상 규칙 기반 태깅 |
| 중급 (1~2개월) | 다양한 양식 대응 + 주석 항목 태깅 |
| 고급 (3개월+) | PDF OCR 연동 + 복합 테이블 구조 자동 인식 |

**자소서 어필 포인트:** 삼일PwC 페이지에서 "수작업의 비중이 매우 높은 업무 성격"을 명시적으로 언급하고 있다. 이 pain point를 정면으로 해결하려는 프로젝트이므로 문제 인식 → 해결 시도의 스토리가 명확하다.

---

### 프로젝트 4. XBRL 공시 변경사항 자동 추적 및 요약 리포트

**방향성:** 분기별 XBRL 공시 파일의 변경사항을 자동으로 추출하고, 변경 내용을 자연어로 요약하는 리포트 생성 도구.

**구체적 구현:**
- 전기/당기 XBRL 파일 비교(diff) 엔진
- 변경 유형 분류 (신규 추가, 삭제, 금액 변동, Taxonomy 변경)
- LLM 기반 변경사항 자연어 요약 리포트 자동 생성

**완성도 기준:**
| 수준 | 설명 |
|------|------|
| MVP (1~2주) | XML diff 기반 변경 항목 리스트업 |
| 중급 (1개월) | 변경 유형 분류 + 중요성 판단 로직 |
| 고급 (2개월+) | GPT/Claude API 연동 자연어 요약 + 시각화 대시보드 |

**자소서 어필 포인트:** Robotic XBRL의 기존 기능 중 "XBRL 파일 간 변화사항 자동 추출"을 AI로 한 단계 고도화하는 프로젝트. 기존 도구를 이해하고 그 위에 가치를 더하는 접근이 실무 감각을 보여준다.

---

### 프로젝트 5. DART 공시 XBRL 데이터 기반 재무 분석 대시보드

**방향성:** DART(전자공시시스템)에서 XBRL 데이터를 수집하여 기업 간 재무 비교 분석을 자동화하는 대시보드. 삼일PwC 페이지에서 언급한 "데이터에 기반한 효과적이고 신속한 의사결정 지원"을 구현한다.

**구체적 구현:**
- DART Open API 활용 XBRL 데이터 자동 수집
- 동일 산업 내 기업 간 주요 재무비율 비교 분석
- 시계열 트렌드 분석 + 이상치 알림

**완성도 기준:**
| 수준 | 설명 |
|------|------|
| MVP (1~2주) | DART API 연동 + 단일 기업 재무제표 조회 |
| 중급 (1개월) | 다수 기업 비교 + 주요 재무비율 자동 산출 |
| 고급 (2~3개월) | 산업별 벤치마크 + 이상치 탐지 + 인터랙티브 대시보드 |

**자소서 어필 포인트:** XBRL 데이터 활용의 최종 목적인 "기업간 재무정보의 비교가능성 제고"를 실제로 구현한 프로젝트. 감사 실무에서 산업 벤치마크 분석에 바로 활용할 수 있어 실용성이 높다.

---

### 프로젝트 6. XBRL Taxonomy 확장항목 관리 및 표준화 제안 시스템

**방향성:** 기업별로 생성되는 확장 Taxonomy 항목을 수집·분석하여 표준화 가능한 항목을 자동 제안하는 시스템. 금감원의 표준화 프로젝트를 지원하는 방향.

**구체적 구현:**
- 다수 기업의 확장항목 수집 및 클러스터링
- 유사 확장항목 그룹핑 + 표준화 후보 추천
- 표준 Taxonomy 개정 시 영향도 분석

**완성도 기준:**
| 수준 | 설명 |
|------|------|
| MVP (2~3주) | 확장항목 수집 + 텍스트 유사도 기반 그룹핑 |
| 중급 (1~2개월) | 산업별 확장항목 패턴 분석 + 표준화 제안 리포트 |
| 고급 (3개월+) | 감독기관용 표준화 권고안 자동 생성 |

**자소서 어필 포인트:** 삼일PwC가 "금융감독원의 비금융업 주석 표준화 프로젝트" 경험을 강조하고 있다. 감독기관 관점의 표준화를 AI로 지원하는 프로젝트는 거시적 시각을 보여준다.

## 3. 전문용어 및 영어약자 용어집

| 용어 | 풀이 | 쉬운 설명 |
|------|------|-----------|
| **XBRL** | eXtensible Business Reporting Language | 기업 재무정보를 컴퓨터가 읽을 수 있는 형태로 만드는 국제 표준 언어. 재무제표를 "데이터화"하는 기술이라고 보면 된다. |
| **iXBRL** | Inline XBRL | 사람이 읽는 재무제표(HTML) 안에 컴퓨터가 읽는 XBRL 태그를 함께 넣는 방식. 하나의 파일로 사람과 기계 모두 읽을 수 있다. |
| **Taxonomy** | 분류체계 | XBRL에서 재무정보를 분류하는 "사전" 같은 것. 예를 들어 "매출액"이라는 항목이 XBRL에서 어떤 코드로 표시되는지 정해놓은 목록이다. |
| **DSD** | Data Structure Definition | XBRL 데이터의 구조를 정의하는 파일. 재무제표의 틀(양식)을 규격화한 설계도와 같다. |
| **ixd** | iXBRL Document | iXBRL 형식으로 작성된 공시 파일. 실제 금감원에 제출하는 전자 파일이다. |
| **Inline** | 인라인 (내장형) | 다른 문서 안에 함께 포함시키는 방식. iXBRL에서 HTML 안에 XBRL 태그를 "내장"한다는 의미. |
| **Tagging / 태깅** | 꼬리표 붙이기 | 재무제표의 각 숫자·항목에 XBRL 표준 코드를 연결하는 작업. 수작업 비중이 높아 자동화 수요가 크다. |
| **Mapping / 매핑** | 대응시키기 | 재무제표 계정과목을 XBRL Taxonomy의 해당 요소에 연결하는 작업. |
| **Validation** | 검증 | XBRL 파일이 규칙에 맞게 작성되었는지 확인하는 과정. 합산 오류, 형식 오류 등을 체크한다. |
| **Calculation Linkbase** | 계산 연결기반 | XBRL에서 숫자 항목 간의 합산 관계를 정의한 파일. 예: 유동자산 + 비유동자산 = 자산총계. |
| **Extension / 확장항목** | 확장 요소 | 표준 Taxonomy에 없는 회사 고유 항목. 예를 들어 특수한 영업 형태의 매출 구분이 표준에 없으면 회사가 직접 만들어 추가한다. |
| **SEC** | Securities and Exchange Commission | 미국 증권거래위원회. 미국 상장사는 SEC에 XBRL 형식으로 재무제표를 제출해야 한다. |
| **DART** | Data Analysis, Retrieval and Transfer System | 금융감독원 전자공시시스템. 한국 상장사의 공시 정보가 모이는 곳이다. |
| **금감원 (FSS)** | Financial Supervisory Service | 금융감독원. 한국의 금융시장을 감독하는 기관으로, XBRL 공시 의무화를 추진하고 있다. |
| **Robotic XBRL** | 로보틱 XBRL | 삼일PwC가 자체 개발한 XBRL 업무 자동화 도구. RPA(로봇 프로세스 자동화) 기술을 활용한다. |
| **RPA** | Robotic Process Automation | 사람이 컴퓨터로 반복하는 작업을 소프트웨어 로봇이 대신 수행하는 기술. |
| **AC** | Acceleration Center | 삼일PwC 내 XBRL 작성기 프로그램 전담 조직. |
| **AX Node** | (삼일PwC 고유 조직명) | XBRL 업무의 디지털화/자동화를 담당하는 삼일PwC 내부 조직. |
| **CRS** | Corporate Reporting Service | 회계기준 전문가 조직. XBRL 표준 Taxonomy 적용에 대한 전문 상담을 제공한다. |
| **NLP** | Natural Language Processing | 자연어 처리. 컴퓨터가 사람의 언어를 이해하고 분석하는 AI 기술 분야. |
| **OCR** | Optical Character Recognition | 광학 문자 인식. 이미지나 PDF 속 글자를 컴퓨터가 읽을 수 있는 텍스트로 변환하는 기술. |
| **LLM** | Large Language Model | 대규모 언어 모델. ChatGPT, Claude 같은 AI. 대량의 텍스트를 학습해 언어를 이해하고 생성한다. |
| **API** | Application Programming Interface | 프로그램끼리 데이터를 주고받는 통로. DART API를 쓰면 공시 데이터를 자동으로 가져올 수 있다. |
| **K-IFRS** | Korean International Financial Reporting Standards | 한국채택국제회계기준. 상장사가 의무적으로 적용하는 회계기준. |
| **Quality** | 품질 | 삼일PwC 페이지에서 "업계 최고의 Quality"라고 표현. XBRL 공시의 정확성과 완성도를 의미. |
| **Simulation** | 시뮬레이션 (모의 실험) | 실제 공시 전에 다양한 시나리오로 미리 테스트해보는 것. |

---

## 4. 자소서 활용 팁

위 6개 프로젝트를 자소서에 녹일 때, **프로젝트 1~3은 삼일PwC XBRL 조직의 핵심 업무(Taxonomy 설계, 품질 검증, 태깅)를 직접 자동화하는 것**이므로 "귀 법인의 XBRL 업무를 깊이 분석하고, 그 pain point를 AI로 해결하려 했다"는 스토리로 쓸 수 있다. 프로젝트 4~6은 기존 도구의 고도화 또는 데이터 활용 확장이므로, 보조적 소재로 활용하되 "현재 → 미래"의 발전 방향을 제시하는 데 쓰면 좋다.

MVP 수준이라도 실제 작동하는 결과물이 있으면 자소서의 설득력이 크게 올라간다. 특히 프로젝트 1(Taxonomy 매핑)이나 프로젝트 5(DART 대시보드)는 공개 데이터만으로 MVP 구현이 가능하므로 진입장벽이 낮다.
